Installation der neuesten ADO Version (ADO 2.5)

Geschrieben von: Christian Koller
Kategorie: Datenbank

Mit dem Erscheinen von Windows 2000 wurde MDAC 2.5 (Microsoft Data Access Components), und damit auch ADO 2.5 ein Bestandteil des Betriebssystems. Wenn sie also ein Windows 2000 Betriebssystem haben, dann ist ADO 2.5 schon automatisch installiert. Um unter Windows 2000 die jeweils neueste ADO Version zu installieren, sind sie auf Service Packs für das Betriebssystem angewiesen, da ein Installieren von ADO (oder MDAC) allein unter Windows 2000 nicht möglich ist.

Um die neueste ADO Version unter älteren Windows Betriebssystemen (Windows 95/98, Windows NT 4.0) zu installieren, laden sie das Setup Program für die neueste MDAC Version (MDAC 2.5 RTM) von der Microsoft Web Site unter http://www.microsoft.com/data/.

Da alle Komponenten von MDAC 2.5 als Teil des Betriebssystems (Windows 2000) gedacht sind, wird eine brauchbare ADO Dokumentation weder mit Windows 2000 noch mit dem MDAC Setup Program mitgeliefert. Stattdessen benötigt man das Microsoft Platform SDK (Software Development Kit) um die vollständige Micorosoft Dokumentation zu MDAC, ADO und verwandten Komponenten zu erhalten.

Installation des Platform SDK

Das Platform SDK kann von der Microsoft Website unter http://msdn.microsoft.com/developer/sdk/platform.asp geladen werden. Achten sie darauf, daß Microsoft Internet Explorer 4.01 (mit Service Pack 2) oder eine neuere Version auf dem Betriebssystem installiert ist, bevor sie das Platform SDK Setup Programm laufen lassen. Für den Fall, dass sie das Platform SDK unter Windows 95 oder Windows 98 installieren benötigen sie möglicherweise zusätzlich den Windows Installer 1.1 (oder neuer).

Wenn sie das Platform SDK Setup Programm starten, können sie über Custom Install auswählen, welche Teile des Platform SDK's sie installieren möchten.

Allgemeines zur Installation von ADO 2.5

Da seit MDAC 2.5 ADO nicht mehr separat installiert werden kann, installieren sie MDAC 2.5 um das darin enthaltene ADO 2.5 mit zu installieren.

Die Setup Prozedur für das neueste MDAC ist wie folgt:

Für Windows 95, Windows 98 und Windows NT 4.0 können sie das neueste MDAC Setup Programm (mdac_typ.exe oder mdacg_typ.exe) von der Microsoft Website unter der Adresse http://www.microsoft.com/data/. laden

Beachten sie, daß MDAC 2.5 (und neuer) nur mehr für x86-kompatible Computer erhältlich ist. Für Alpha Maschinen benützen sie bitte MDAC 2.1 (SP2).

Unter Windows 2000 ist MDAC 2.5 ein integraler Bestandteil des Betriebssystems. Da Windows 2000 das Windows File Protection (WFP) System benutzt um alle Betriebssystemdateien vor dem Löschen oder Überschreiben zu schützen, kann eine neue MDAC Version nicht einfach installiert werden wie auf den alten Windows Betriebssystemen. Anstatt das MDAC Setup Programm zu verwenden, müssen sie das neueste Service Pack für Windows 2000 installieren.

MDAC Setup Programm

Gehen sie die folgende Checkliste durch um das MDAC Setup Prgramm (mdac_typ.exe oder mdacg_typ.exe) erfolgreich laufen zu lassen.

  1. Laden sie die neueste MDAC Version aus dem Internet. Die Adresse für das MDAC Setup Programm lautet http://www.microsoft.com/data/
  2. Wenn sie MDAC unter Windows 95 installieren möchten, dann vergewissern sie sich, daß DCOM95 auf dem Computer verfügbar ist. Wenn sie DCOM95 nicht haben so laden sie es von der Microsoft Website unter der Adresse http://www.microsoft.com/com/resources/downloads.asp herunter und installieren sie es.
  3. Unter Windows 98 is DCOM98 für gewöhnlich installiert. Falls sie DCOM98 benötigen, sie finden es unter derselben Internet Adresse wie DCOM95.
  4. Um MDAC 2.5 benutzen zu können benötigen sie mindestens Internet Explorer 4.01 (SP 2) oder neuer. Andernfalls können Teile des MDAC nicht benutzt werden da sie auf Resourcen des Internet Explorer zugreifen.
  5. Bevor sie das MDAC Setup Programm (mdac_typ.exe) laufen lassen, beenden sie alle Windows Applikationen und Windows NT Services die MDAC benutzen könnten. Für optimale Resultate beenden sie alle Applikationen und Programme, speziell Entwicklungstools und Server Applikationen.
  6. Wenn sie MDAC auf einer Windows NT 4.0 Maschine installieren, so loggen sie sich zuvor als Administrator ein.
  7. Beenden sie alle Anti-Virus, Harddisk Mangement Programme und speicherresistente Applikationen die noch auf dem Computer laufen.
  8. Die MDAC Installation benötigt mindestens 50 MB freien Festplattenspeicher. Prüfen sie daher den verfügbaren Festplattenspeicher. Beachten sie, daß MDAC seine Dateien sowohl in den "Programme" Ordner als auch in den System Ordner (Windows\System unter Windows 9x, oder WinNT\System32 unter Windows NT) kopiert.
  9. Stellen sie sicher, dass der TEMP Ordner eingerichtet und das zugehörige Laufwerk mindestens 50 MB Speicher frei hat. Sie können mittles des SET Kommandos (in einem DOS oder Command Prompt ausgeführt) die sogenannten Umgebungsvariablen des Betriebssystems ausgeben lassen, wobei der ausgegebene Wert der Variablen TEMP= und TMP= den Pfad des TEMP Ordners angeben.
  10. Wenn sie das MDAC Setup Programm unter Windows 95 ausführen wollen, so muss sich das Setup Programm auf dem Computer oder einem eingemappten Laufwerk befinden.
  11. Für den Fall, dass sie MDAC auf einem Windows NT 4.0 Terminal Server installieren wollen, müssen sie eine spezielle Vorgehensweise beachten. Weitere Informationen sind im Microsoft Knowledge Base Artikel mir der Nummer Q216149 enthalten.
  12. Starten sie das MDAC Setup Programm mdac_typ.exe auf dem Rechner. Sollte eine Nachricht erscheinen, daß DCOM95 oder DCOM98 benötigt wird, so installieren sie zuerst DCOM 95 (dcom95.exe) oder DCOM98 (dcom98.exe) und starten danach das MDAC Setup Programm erneut.
  13. Für den Fall, daß sie MDAC 2.5 (oder neuer) auf einem Computer installieren, der auf eine SQL Server 6.5 Datenbank zugreift, haben sie noch folgendes zu beachten. Bei einem veraltertem Satz von Catalog Stored Procedures auf dem SQL Server wird bei einem Zugriff unter MDAC 2.5 eine Warnung ausgegeben. Neben dieser Warnung kann es unter MDAC 2.5 zu ernsthaften Problemen kommen, wie zum Beispiel dem Verlust von Information über den Aufbau einer Tabelle. Um dieses Problem zu lösen müssen sie die SQL Scripts der Datei names instcat.sql auf dem SQL Server laufen lassen. Die instcat.sql Datei ist im MDAC 2.5 enthalten. Genausogut können sie auch das Service Pack 5 (oder neuer) für SQL Server 6.5 auf dem Datenbank Server installieren, wobei die neuen Catalog Stored Procedures automatisch angelegt werden.

Wenn sie eine ältere MDAC Version über eine neuere spielen möchten, so müssen sie zuerst das installierte MDAC vom Computer entfernen. Da MDAC ohne De-Installationsroutine ausgeliefert wird, müssen sie zum De-Installierern auf das Component Checker Tool zurückgreifen, welches sie von der Microsoft Site unter http://www.microsoft.com/Data/download.htm herunterladen können.

Durch die Installation einer neuen MDAC Version auf ihrem Computer werden alle ADO Komponenten, OLE DB Provider und ODBC Treiber von Microsoft Produkten auf einen einheitlichen neuen Stand gebracht. Um OLE DB Provider und ODBC Treiber anderer Hersteller upzudaten, müssen sie diese nach der Installation von MDAC gesondert installieren.

Das MDAC Setup Programm ist geeignet um es gemeinsam mit einer eigenen Applikation zu vertreiben. So bereiten sie einen Kundencomputer optimal für eine Datenbank Applikation vor. Beachten sie bitte, daß MDAC nur auf Betriebssystemen installiert werden darf, wenn das Betriebssystem gültig lizensiert ist (siehe zum Beispiel MDAC 2.5 End-User License Agreement, kurz EULA).

MDAC Setup Programme (mdac_typ.exe) zum Weitervertrieb können von der Microsoft Internet Site unter http://www.microsoft.com/data/ heruntergeladen werden. Um MDAC 2.0 (oder neuer) auf Windows 95 (oder Windows 98) zu installieren benötigt das Betriebssystem das DCOM95 (oder DCOM98) Update, welches sie auch bei Microsoft auf der Website zum Download finden.

Überprüfen der MDAC Installation

Das Beste, was sie zur Überprüfung einer MDAC Installation tun können, ist eine Datenbank Applikation laufen zu lassen, die ADO und andere MDAC Komponenten benutzt.

Auch eine Analyse des Systems mit dem Component Checker Tool stellt weitgehend sicher, daß MDAC und ADO korrekt installiert sind.
