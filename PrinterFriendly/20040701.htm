Visual Studio 2005 und .NET 2.0 Beta 1

Geschrieben von: Christoph Wille
Kategorie: .NET 2.0

This printed page brought to you by AlphaSierraPapa

Am 29ten Juni hat Microsoft eine Welle von Betaprodukten auf die Entwicklergemeinde losgelassen. Um sozusagen für die Sommerbeschäftigung (und darüber hinaus) zu sorgen, habe ich in diesem Artikel wichtige Anlaufpunkte für den "verspielten" Programmierer zusammengetragen, um das Verwenden der Betas auch zum Vergnügen werden zu lassen.

Das wichtigste vorweg: dieser Artikel ist mit Absicht kurz gehalten. Sein Hauptzweck ist es, Verweise in das AspxFiles Wiki zu legen: diese sind quasi tagesaktuell gewartet, und es ist auch jeder eingeladen, dort aktiv Inhalte einzutragen. Diese Art der Interaktion ist mit einem (statischen) Artikel alleine einfach nicht zu erreichen.

Visual Studio 2005

Beginnen wir mit dem Thema, das wahrscheinlich die meisten am brennendsten interessiert - ja, es kann downgeloadet werden, von jedem, nicht nur eine kleine Gemeinde von Beta Usern. Zur "klassischen" Version gibt es im Wiki die Sektion Visual Studio 2005.

Warum "klassisch"? Nun, es gibt eine neue Schiene, die sich Visual Studio 2005 Express nennt. Dies sind Ableger der klassischen Version, die darauf ausgerichtet sind, daß der Programmierer schneller produktiv wird, ohne sich zuerst lange mit dem Erlernen der Bedienung der Entwicklungsumgebung (und auch Teilen der Infrastruktur) abmühen zu müssen.

Speziell interessant unter den Express Produkten sind Visual Web Developer 2005 Express Edition und die Datenbank SQL Server 2005 Express. Dazu wird es später auch hier Artikel geben.

Zu guter letzt ist noch Visual Studio 2005 Team System zu nennen, das aber leider im Moment noch nicht für jedermann zum Download & Testen zur Verfügung steht.

Das 2.0 Framework

Mit Visual Studio 2005 bekommt man das Framework bereits mitgeliefert, allerdings gibt es im Wiki auch noch zusätzliche Sektionen für ASP.NET 2.0, C# 2.0 (Generics, etc) und natürlich auch das Framework im Allgemeinen.

Deutsche Mailingliste

Es gibt eine Whidbey Mailingliste, auf der Early Adopter sich auf Deutsch austauschen können. Etliche der Listenmitglieder arbeiten bereits seit frühen Alpha-Tagen mit Whidbey, daher dürfte selten eine Frage unbeantwortet bleiben.

Schlußbemerkung

Dieser Artikel - besser gesagt Linkliste zu tagesaktuellem Content - soll den Entwicklern helfen einen schnellen Start in die Technologie zu finden, die nächstes Jahr als Release wahrscheinlich sehr schnell den Weg auf unsere Entwicklermaschinen findet: ein kleiner "Frühstart" ist somit kein Fehler.
